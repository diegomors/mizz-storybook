# Guia de Uso do Design System

Este guia fornece instruções práticas sobre como utilizar o Mizz Design System em seus projetos.

## 📦 Instalação

Os componentes estão localizados em `/src/app/components/mizz/` e podem ser importados diretamente:

```tsx
import { MizzButton } from './components/mizz/MizzButton';
import { MizzTextField } from './components/mizz/MizzTextField';
import { MizzCard } from './components/mizz/MizzCard';
```

## 🎨 Usando as Cores

As cores do design system estão definidas como valores hexadecimais. Use-as diretamente no Tailwind:

```tsx
// Cor primária (Laranja Mizz)
<div className="bg-[#ff5c00] text-white">
  Elemento com cor primária
</div>

// Cor de texto
<p className="text-[#ff5c00]">
  Texto em laranja
</p>

// Bordas
<div className="border-2 border-[#ff5c00]">
  Borda laranja
</div>
```

### Cores Semânticas

```tsx
// Sucesso
<div className="bg-[#00d68f]">Sucesso</div>

// Erro
<div className="bg-[#ff4949]">Erro</div>

// Aviso
<div className="bg-[#ffa726]">Aviso</div>
```

## ✏️ Usando a Tipografia

A fonte **Be Vietnam Pro** está disponível no projeto. Use-a com `font-family`:

```tsx
// Título grande
<h1 className="text-[96px] font-extrabold" style={{ fontFamily: 'Be Vietnam Pro' }}>
  Título Principal
</h1>

// Texto de corpo
<p className="text-[20px]" style={{ fontFamily: 'Be Vietnam Pro' }}>
  Texto de corpo regular
</p>

// Negrito
<p className="font-bold" style={{ fontFamily: 'Be Vietnam Pro' }}>
  Texto em negrito
</p>
```

### Tamanhos Recomendados

| Elemento | Tamanho | Peso |
|----------|---------|------|
| Página principal | 96px | ExtraBold |
| Seções principais | 64px | ExtraBold |
| Títulos de cards | 32px | Bold |
| Corpo de texto | 20px | Regular |
| Legendas | 14px | Regular |

### Formulário Completo

```tsx
import { useState } from 'react';
import { MizzTextField } from './components/mizz/MizzTextField';
import { MizzRadioGroup } from './components/mizz/MizzRadioGroup';
import { MizzButton } from './components/mizz/MizzButton';

function Form() {
  const [name, setName] = useState('');
  const [payment, setPayment] = useState('pix');

  return (
    <form className="space-y-6 max-w-md">
      <MizzTextField
        label="Nome completo"
        placeholder="Digite seu nome"
        value={name}
        onChange={(e) => setName(e.target.value)}
      />
      
      <MizzRadioGroup
        name="payment"
        label="Forma de pagamento"
        options={[
          { value: 'pix', label: 'PIX' },
          { value: 'credit', label: 'Cartão de crédito' },
          { value: 'cash', label: 'Dinheiro' },
        ]}
        value={payment}
        onChange={setPayment}
      />
      
      <MizzButton variant="primary" size="lg" className="w-full">
        Finalizar
      </MizzButton>
    </form>
  );
}
```

## 🎯 Boas Práticas

### 1. Consistência

Use sempre os mesmos componentes para ações similares:

✅ **Correto**
```tsx
<MizzButton variant="primary">Confirmar</MizzButton>
<MizzButton variant="primary">Enviar</MizzButton>
<MizzButton variant="primary">Finalizar</MizzButton>
```

❌ **Evite**
```tsx
<MizzButton variant="primary">Confirmar</MizzButton>
<button className="bg-blue-500">Enviar</button>
<a className="bg-green-500">Finalizar</a>
```

### 2. Hierarquia Visual

Use os tamanhos e pesos corretos para criar hierarquia:

```tsx
<h1 className="text-5xl font-extrabold">Título Principal</h1>
<h2 className="text-3xl font-bold">Subtítulo</h2>
<p className="text-lg">Texto de corpo</p>
<p className="text-sm text-gray-600">Legenda</p>
```

### 3. Estados de Componentes

Sempre forneça feedback visual para interações:

```tsx
// Estado normal, hover, loading e disabled
<MizzButton variant="primary">Normal</MizzButton>
<MizzButton variant="primary" loading>Carregando</MizzButton>
<MizzButton variant="primary" disabled>Desabilitado</MizzButton>

// Estados de erro
<MizzTextField 
  label="Email" 
  error="Email inválido"
/>
```

### 4. Acessibilidade

Todos os componentes Mizz já possuem **acessibilidade integrada**:

- ✅ Labels associados automaticamente via IDs únicos
- ✅ ARIA attributes apropriados (`aria-invalid`, `aria-required`, `aria-busy`)
- ✅ Navegação por teclado funcional
- ✅ Estados de foco visíveis
- ✅ Mensagens de erro com `role="alert"`
- ✅ Proteção contra múltiplos cliques em botões

```tsx
// ✅ Acessível por padrão
<MizzTextField 
  label="Email"
  placeholder="seu@email.com"
  required
  error="Email inválido"
/>

// ✅ Botão com loading automático
<MizzButton 
  variant="primary"
  onClick={async () => {
    await saveData(); // Automaticamente mostra loading
  }}
>
  Salvar
</MizzButton>
```

### 5. Controle de Integridade

Os componentes acionáveis possuem **proteção integrada**:

```tsx
// Proteção automática contra cliques duplos
<MizzButton 
  variant="primary"
  onClick={async () => {
    // Durante a execução, o botão fica desabilitado automaticamente
    await processPayment();
  }}
>
  Processar pagamento
</MizzButton>

// Loading manual quando necessário
<MizzButton variant="primary" loading>
  Processando...
</MizzButton>
```

## 💡 Dicas

1. **Sempre use os componentes do design system** ao invés de criar custom components
2. **Teste em diferentes tamanhos de tela** para garantir responsividade
3. **Valide acessibilidade** com ferramentas como Lighthouse (os componentes já são acessíveis!)
4. **Mantenha consistência** em toda a aplicação
5. **Consulte o Storybook** quando tiver dúvidas sobre como usar um componente
6. **Use os code snippets** para acelerar o desenvolvimento
7. **Aproveite os estados de loading automáticos** em botões
